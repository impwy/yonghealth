# YongHealth - 수정 계획서 v2

기존 수정 계획서(requirement.md)를 기준으로, 아래 추가 변경 요구사항을 반영한 수정안이다.

- 네비바에 **쉬는 시간 타이머** 기능 추가
- **UI 디자인 개선** — 전반적인 시각적 완성도 향상
- 운동 종목 선택 UI **레이어 겹침 버그 수정**

---

## 1. 변경 요구사항

### 1.1 기능 요구사항

#### FR-7. 쉬는 시간 타이머 (Rest Timer)

- 네비바(상단 또는 하단 내비게이션)에 타이머 진입점을 제공해야 한다.
- 타이머는 **카운트다운** 방식을 기본으로 한다.
  - 기본 쉬는 시간 설정값 제공 (예: 60초, 90초, 120초 프리셋)
  - 사용자가 직접 시간 조정 가능해야 한다 (+ / - 버튼 또는 슬라이더)
- 타이머 동작 중에는 네비바에 남은 시간이 표시되어야 한다.
- 타이머 종료 시 사용자에게 알림을 주어야 한다.
  - 진동(모바일), 소리, 또는 시각적 플래시 중 하나 이상 지원
- 운동 기록 화면을 이탈해도 타이머는 계속 동작해야 한다 (화면 전환 유지).
- 타이머는 일시정지 및 리셋이 가능해야 한다.

#### FR-8. UI 디자인 개선

- 현재 디자인은 지나치게 단순하므로, 시각적 완성도를 높여야 한다.
- 다음 영역에 디자인 개선을 적용한다:
  - **달력 화면**: 날짜 셀 스타일링, 운동 기록 있는 날 강조 표시 (색상, 점, 배지)
  - **운동 세션 카드**: 시간, 종목 수, 세트 수를 시각적으로 구분
  - **세트 입력 영역**: 단순 테이블에서 카드형/인라인 입력으로 개선
  - **버튼 및 CTA**: 색상 계층 구분 (Primary / Secondary / Danger)
  - **빈 상태(Empty State)**: 운동 기록 없는 날에 일러스트 또는 안내 문구 제공
  - **로딩 상태**: 스켈레톤 UI 또는 스피너로 콜드스타트 대기 시간 체감 완화
- 전체 색상 테마를 통일한다 (예: 포인트 컬러 1개 + 중립 컬러 계열).
- 폰트 크기 계층(heading / body / caption)을 명확히 구분한다.

### 1.2 버그 수정

#### BUG-1. 운동 종목 선택 시 전체 탭 목록 겹침

- **현상**: 운동 종목 추가 화면에서 종목을 선택할 때, 전체 탭의 목록 UI가 선택 드롭다운/시트와 겹쳐 보이는 문제 발생
- **예상 원인**:
  - `ExercisePicker` 컴포넌트의 `z-index` 또는 `position` 설정 누락
  - 부모 컨테이너의 `overflow: hidden`이 드롭다운을 클리핑
  - 모달/시트 레이어보다 낮은 쌓임 맥락(stacking context)
- **수정 방향**:
  - 종목 선택 UI를 페이지 내 인라인이 아닌 **Bottom Sheet 또는 전용 모달**로 분리
  - 또는 `z-index` 계층을 명확히 정의하고 `portal` 방식으로 DOM 최상단에 렌더링
  - 탭 컨테이너의 `overflow` 속성 재검토

## 2. 컴포넌트 변경안

### 2.1 타이머 관련 신규 컴포넌트

```text
frontend/components/
├── RestTimer.tsx              (타이머 핵심 로직 + UI)
├── RestTimerButton.tsx        (네비바 진입점 — 남은 시간 표시)
└── ui/
    └── TimerSheet.tsx         (타이머 조작 패널 — 하단 시트)
```

### 2.2 UI 개선 대상 기존 컴포넌트

```text
frontend/components/
├── WorkoutCalendar.tsx        (날짜 셀 스타일, 운동 여부 배지)
├── WorkoutCard.tsx            (세션 카드 디자인)
├── ExerciseAccordion.tsx      (종목 아코디언 시각 개선)
├── ExercisePicker.tsx         (BUG-1 수정 + 선택 UI 개선)
├── SetTable.tsx               (세트 입력 카드형 전환)
└── ui/
    ├── Navbar.tsx             (타이머 버튼 추가)
    ├── BottomNav.tsx          (타이머 진입점 배치)
    ├── EmptyState.tsx         (신규 — 빈 상태 컴포넌트)
    └── Skeleton.tsx           (신규 — 로딩 스켈레톤)
```

## 3. 비기능 요구사항

#### NFR-4. 타이머 동작 안정성
- 탭 전환, 페이지 이동 시에도 타이머 상태가 유지되어야 한다.
- 타이머 상태는 전역 상태(Context 또는 Zustand 등)로 관리하거나 `sessionStorage`로 복원 가능해야 한다.
- 백그라운드 탭에서도 타이머가 정확히 동작해야 한다 (`visibilitychange` 이벤트 고려).

#### NFR-5. 디자인 일관성
- Tailwind CSS 커스텀 색상 팔레트를 `tailwind.config.ts`에 정의하여 전체 일관성 확보.
- 반복 사용되는 UI 패턴은 재사용 가능한 컴포넌트로 분리.

## 4. 구현 순서 (기존 Phase 이후 추가)

### Phase 10: 쉬는 시간 타이머
1. `RestTimer` 컴포넌트 구현 (카운트다운 로직)
2. 전역 타이머 상태 관리 설정 (Context 또는 상태 라이브러리)
3. 네비바에 `RestTimerButton` 추가 (남은 시간 표시)
4. `TimerSheet` 구현 (프리셋 선택, +/- 조작, 일시정지/리셋)
5. 종료 알림 구현 (진동 or 소리 or 시각 효과)

### Phase 11: BUG-1 수정 — 종목 선택 겹침
1. `ExercisePicker` 렌더링 방식 분석
2. Bottom Sheet 또는 Portal 방식으로 전환
3. `z-index` / `overflow` 스타일 정리
4. 모바일/데스크탑 양쪽에서 재현 테스트

### Phase 12: UI 디자인 개선
1. Tailwind 커스텀 색상 팔레트 정의
2. 달력 셀 스타일링 (운동 기록 배지, 오늘 날짜 강조)
3. 세션 카드 디자인 개선
4. 세트 입력 카드형 전환
5. 버튼 계층 색상 적용 (Primary / Secondary / Danger)
6. 빈 상태 컴포넌트 (`EmptyState`) 구현
7. 로딩 스켈레톤 (`Skeleton`) 구현
